01What is it?

A single-page website that shows every major sports game happening today, with live scores, betting odds, weather, lineups, and ticket prices — all on one screen.

Live Scores
Auto-refresh every 30 seconds
Favorite Teams
Star teams; they float to the top
Filters & Search
By league, by team, live-only
Game Details Modal
Lineups · Odds · Tickets · Weather

05Data flow — from internet to your eyes

When the page loads, data travels through five stages, left to right.

Source
ESPN / Odds API / SeatGeek / Open-Meteo
Public sports data on the internet.
Service
sportsApi.ts & friends
Fetches the raw JSON. Normalizes it to our shape. Falls back to mock data if offline.
Hook
useGames / useFilters
Stores the result in React state. Re-runs every 30 s for live games. Applies filters.
Page
Dashboard.tsx
Decides the layout: grid vs list, today vs upcoming, with/without standings panel.
UI
GameCard · ScoreTicker · Modal
Renders the data you actually see and click on.

Favorites travel in reverse: you click a star → useFavorites updates → it writes to your browser's localStorage so it survives refresh.

07A click, step by step

What actually happens from the moment you open the URL to the moment you star a favorite team.

  1. Browser loads index.html
    It's nearly empty — just a <div id="root">. Then it loads main.tsx.
  2. main.tsx mounts <App /> into that div
    App.tsx simply renders <Dashboard /> — the real page.
  3. Dashboard calls useGames(...)
    "Hey, I need today's games." This kicks off a fetch.
  4. sportsApi.ts calls ESPN — 8 leagues in parallel
    NBA, NFL, MLB, NHL, NCAAF, NCAAB, MLS, EPL. It normalizes every response into our universal Game shape.
  5. Odds + tickets + weather merge in
    From oddsApi.ts, ticketsApi.ts, weatherApi.ts. If no API key is set, mock data fills in.
  6. useFilters trims the list
    If you have league filters or "favorites only" turned on, only matching games make it to the screen.
  7. Dashboard renders one <GameCard /> per remaining game
    Sport-specific layout chosen automatically per league.
  8. Every 30 seconds, useGames silently re-fetches
    Live scores stay current. The page never reloads — only the data does.
  9. You click a game → setSelectedGame() runs
    The GameDetailsModal opens with venue, lineups, odds table, tickets, weather.
  10. You click the ⭐ on a team
    useFavorites.toggleFavorite() updates state AND writes to localStorage. Survives refresh.

08The tech stack in plain English

The five tools that make this project work.

React
Library for building the page out of reusable Lego-like pieces called components.
TypeScript
Spell-check for code. Catches mistakes before the page ever runs.
Vite
Bundles everything and runs the local dev server you preview in the browser.
Tailwind CSS
Styling via tiny pre-made classes (e.g., text-red-500) instead of custom CSS.
Axios
Small helper that fetches data from other websites (the APIs).

09How to run it

From the terminal, inside this project folder:

# Install everything (only needed the first time) npm install # Start the dev server. Open the URL it prints in your browser. npm run dev # Build a production bundle (output appears in dist/) npm run build # Preview that production bundle locally npm run preview
